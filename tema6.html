Programación Orientada a Objetos (POO)

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el código en torno a "objetos", los cuales contienen datos (atributos) y métodos (funciones) que operan sobre esos datos. Java es un lenguaje orientado a objetos, y todo su código debe estar dentro de clases.

Conceptos fundamentales

  • Clase: Es una plantilla o molde que define los atributos y métodos que tendrán los objetos de ese tipo. Por ejemplo, la clase Coche define que todo coche tiene marca, modelo y color, y puede arrancar y frenar.
  • Objeto: Es una instancia concreta de una clase. Por ejemplo, un coche específico: "Ford Focus rojo".
  • Atributo (o campo): Variable que pertenece a la clase y representa una característica del objeto.
  • Método: Función definida dentro de la clase que describe un comportamiento de los objetos.

Declaración de una clase

Una clase se define con la palabra clave class seguida del nombre con un formato CamelCase (Primera letra en mayúscula seguida de minuscula). Dentro de las llaves se colocan los atributos y métodos.

public class Persona {
    // Atributos (variables de instancia)
    String nombre;
    int edad;
    
    // Métodos
    public void saludar() {
        System.out.println("Hola, soy " + nombre);
    }
}

En este ejemplo, hemos definido una clase Persona con dos atributos (nombre y edad) y un método saludar() que imprime un saludo.

Importante: Por ahora los atributos no tienen modificador de acceso, lo que significa que son package-private (accesibles desde cualquier clase del mismo paquete). Más adelante veremos encapsulamiento con private.

Creación de objetos (instanciación)

Para crear un objeto (instancia) de una clase se usa el operador new seguido del constructor de la clase. Una vez creado, podemos acceder a sus atributos y métodos usando el operador punto ..

public class EjemploPersona {
    public static void main(String[] args) {
        // Crear un objeto de la clase Persona
        Persona persona1 = new Persona();
        
        // Asignar valores a los atributos
        persona1.nombre = "Ana";
        persona1.edad = 25;
        
        // Llamar a un método
        persona1.saludar();  // Imprime: Hola, soy Ana
        
        // Podemos crear otro objeto independiente
        Persona persona2 = new Persona();
        persona2.nombre = "Carlos";
        persona2.edad = 30;
        persona2.saludar();  // Imprime: Hola, soy Carlos
    }
}

En este ejemplo, hemos creado dos objetos de la clase Persona, cada uno con sus propios valores para los atributos. Al llamar al método saludar(), cada objeto muestra su propio nombre.

Cada objeto tiene sus propios valores para los atributos. Modificar los atributos de un objeto no afecta a los demás.

Constructores

Un constructor es un método especial que se ejecuta automáticamente al crear un objeto con new. Su misión es inicializar el objeto, normalmente asignando valores iniciales a los atributos. Tiene el mismo nombre que la clase y no devuelve ningún valor (ni siquiera void).

Constructor por defecto

Si no definimos ningún constructor, Java proporciona uno por defecto sin parámetros que no hace nada (los atributos se inicializan con valores por defecto: null para objetos, 0 para números, false para booleanos).

Constructor con parámetros

Podemos definir nuestros propios constructores para inicializar los atributos con valores específicos.

public class Persona {
    String nombre;
    int edad;
    
    // Constructor con parámetros
    public Persona(String nombre, int edad) {
        this.nombre = nombre;  // this.nombre se refiere al atributo, nombre al parámetro
        this.edad = edad;
    }
    
    public void saludar() {
        System.out.println("Hola, soy " + nombre + " y tengo " + edad + " años.");
    }
}

// Uso en main:
Persona p = new Persona("Luis", 22);
p.saludar();

En este ejemplo, el constructor de la clase Persona recibe dos parámetros para inicializar los atributos nombre y edad. Al crear un nuevo objeto con new Persona("Luis", 22), se asignan esos valores a los atributos del objeto.

De este modo, cada objeto creado con el constructor tiene sus propios valores para los atributos. Y por consiguiente, no puede haber un objeto con el valor nulo para los atributos ya que se inicializan con valores específicos.

La palabra clave this

this es una referencia al objeto actual. Se usa para distinguir entre los parámetros del constructor y los atributos de la clase cuando tienen el mismo nombre. También se puede usar para llamar a otro constructor de la misma clase (con this(...)).

Sobrecarga de constructores

Al igual que los métodos, los constructores pueden sobrecargarse, es decir, tener varios con distintos parámetros.

public class Persona {
    String nombre;
    int edad;
    
    // Constructor sin parámetros (por si queremos crear persona y luego asignar)
    public Persona() {
        this.nombre = "Desconocido";
        this.edad = 0;
    }
    
    public Persona(String nombre) {
        this.nombre = nombre;
        this.edad = 18;  // edad por defecto
    }
    
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }
}

En este ejemplo, la clase Persona tiene tres constructores: uno sin parámetros, otro con un solo parámetro y otro con dos parámetros. Esto nos da flexibilidad para crear objetos de diferentes formas según nuestras necesidades.

Encapsulamiento: modificadores de acceso

El encapsulamiento es uno de los pilares de la POO. Consiste en ocultar los detalles internos de un objeto y permitir el acceso solo a través de métodos controlados. Para ello se usan los modificadores de acceso:

Modificador Accesible desde
private Solo dentro de la misma clase.
public Desde cualquier clase.
protected Dentro del mismo paquete y subclases (lo veremos más adelante).
(sin modificador) Package-private: accesible solo desde clases del mismo paquete.

Lo habitual es declarar los atributos como private y proporcionar métodos públicos getters (para obtener el valor) y setters (para modificarlo, si es necesario).

public class Persona {
    private String nombre;
    private int edad;
    
    // Constructor
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        setEdad(edad);  // usamos el setter para validar
    }
    
    // Getter para nombre
    public String getNombre() {
        return nombre;
    }
    
    // Setter para nombre
    public void setNombre(String nombre) {
        this.nombre = nombre;
    }
    
    // Getter para edad
    public int getEdad() {
        return edad;
    }
    
    // Setter para edad con validación
    public void setEdad(int edad) {
        if (edad >= 0 && edad <= 150) {
            this.edad = edad;
        } else {
            System.out.println("Edad no válida");
        }
    }
    
    public void saludar() {
        System.out.println("Hola, soy " + nombre);
    }
}

En este ejemplo, los atributos nombre y edad están declarados como private, lo que significa que no pueden ser accedidos directamente desde fuera de la clase. Para acceder a ellos, se utilizan métodos públicos llamados getters y setters.

Beneficios del encapsulamiento:

  • Controlamos cómo se accede y modifica los datos (podemos añadir validaciones).
  • Podemos cambiar la implementación interna sin afectar a quienes usan la clase.
  • Ocultamos la complejidad y exponemos solo lo necesario.

Métodos de instancia

Los métodos de instancia son aquellos que no tienen la palabra clave static y operan sobre los atributos de un objeto concreto. Pueden acceder directamente a los atributos (incluso privados) y llamar a otros métodos de instancia.

public class Calculadora {
    private double memoria;
    
    public void sumar(double valor) {
        memoria += valor;
    }
    
    public void restar(double valor) {
        memoria -= valor;
    }
    
    public double getMemoria() {
        return memoria;
    }
    
    public void reset() {
        memoria = 0;
    }
}

En este ejemplo, la clase Calculadora tiene un atributo privado memoria y varios métodos de instancia para sumar, restar, obtener el valor de memoria y resetearla.

Para usar estos métodos necesitamos un objeto:

Calculadora calc = new Calculadora();
calc.sumar(10);
calc.sumar(5);
System.out.println(calc.getMemoria()); // 15

En este caso, hemos creado un objeto calc de la clase Calculadora y hemos llamado a sus métodos para realizar operaciones. Cada objeto de la clase Calculadora tendrá su propia memoria independiente.

Ejemplo completo: Clase Coche

Vamos a crear una clase Coche con atributos privados, constructores, getters/setters y un método que muestre información.

public class Coche {
    private String marca;
    private String modelo;
    private int velocidad;  // km/h
    
    // Constructor por defecto
    public Coche() {
        this.marca = "Desconocida";
        this.modelo = "Desconocido";
        this.velocidad = 0;
    }
    
    // Constructor con parámetros
    public Coche(String marca, String modelo) {
        this.marca = marca;
        this.modelo = modelo;
        this.velocidad = 0;
    }
    
    // Getters y Setters
    public String getMarca() {
        return marca;
    }
    
    public void setMarca(String marca) {
        this.marca = marca;
    }
    
    public String getModelo() {
        return modelo;
    }
    
    public void setModelo(String modelo) {
        this.modelo = modelo;
    }
    
    public int getVelocidad() {
        return velocidad;
    }
    
    // Métodos de comportamiento
    public void acelerar(int incremento) {
        if (incremento > 0) {
            velocidad += incremento;
            System.out.println("Acelerando... Velocidad actual: " + velocidad + " km/h");
        }
    }
    
    public void frenar(int decremento) {
        if (decremento > 0 && velocidad - decremento >= 0) {
            velocidad -= decremento;
            System.out.println("Frenando... Velocidad actual: " + velocidad + " km/h");
        } else {
            System.out.println("No se puede frenar tanto");
        }
    }
    
    public void mostrarInfo() {
        System.out.println("Coche: " + marca + " " + modelo + ", velocidad: " + velocidad + " km/h");
    }
}

En este ejemplo, la clase Coche tiene atributos privados y métodos públicos para manipularlos. Cada objeto de la clase Coche tiene sus propios valores para los atributos.

Programa principal que usa la clase

public class TestCoche {
    public static void main(String[] args) {
        Coche coche1 = new Coche("Ford", "Focus");
        coche1.mostrarInfo();  // Coche: Ford Focus, velocidad: 0 km/h
        
        coche1.acelerar(50);
        coche1.acelerar(20);
        coche1.frenar(30);
        coche1.mostrarInfo();  // Coche: Ford Focus, velocidad: 40 km/h
        
        // Uso de setter para cambiar la marca
        coche1.setMarca("Seat");
        coche1.mostrarInfo();  // Coche: Seat Focus, velocidad: 40 km/h
        
        // Crear otro coche con constructor por defecto
        Coche coche2 = new Coche();
        coche2.mostrarInfo();  // Coche: Desconocida Desconocido, velocidad: 0 km/h
    }
}

En este programa principal, hemos creado un objeto coche1 de la clase Coche usando el constructor con parámetros, y luego hemos llamado a sus métodos para mostrar información, acelerar y frenar. También hemos creado otro objeto coche2 usando el constructor por defecto.
